# OS의 구성

OS를 큰 개념으로 바라보면 핵심이 되는 커널과 시스템 프로그램 그리고 응용 프로그램으로 구성된다.

OS의 구성 방식은 OS가 제공하는 서비스를 어떤 요소가 어떤 방식으로 제공하는지와 연관이 깊다.

## 서비스 관점에서의 OS

OS는 자원 할당자, 제어 프로그램 등의 역할을 하는데 제공하는 서비스를 컴퓨터 시스템 구성 요소와 함께 보면 다음과 같다.

![OS 서비스](img/os-service.svg)

- 사용자 인터페이스
  - GUI, CLI, Batch 방식등으로 사용자가 컴퓨터 시스템에 명령을 내리기 위한 환경을 제공한다.
- 시스템 콜
  - OS가 제공하는 서비스에 대한 인터페이스로 OS의 서비스를 안전하게 제공받기 위한 인터페이스를 제공한다.
- 프로그램 실행
  - 프로그램을 메모리에 적재해 실행할 수 있는 환경을 제공한다.
- I/O 연산
  - I/O 장치 등을 활용하는 I/O 연산의 처리 수단을 제공한다.
- 파일 시스템
  - 파일 조작이 필요한 프로그램에게 파일의 생성, 삭제, 검색, 권한 처리 등의 기능을 제공한다.
- 통신
  - 동일 컴퓨터 또는 네트워크 상의 여러 프로세스 간에 정보를 교환할 수 있는 환경을 제공한다.
- 보호, 보안
  - 다중 사용자 또는 네트워크 환경에서 정보의 사용을 통제한다.
  - 프로세스가 OS 자체를 방해하지 않도록 한다.
- 오류 탐지
  - 하드웨어 오류, 프로그램 오류에 대한 적절한 조치를 제공한다.
- 로깅
  - 시스템의 자원 사용량을 기록, 관리한다.
- 자원 할당
  - 다수의 프로세스, 작업이 동시에 실핼될 때 문제가 없도록 적절히 자원을 할당해준다.

## OS 구성 방식

OS의 구성 방식은 핵심인 커널과 인터페이스, 장치 드라이버 등을 어떻게 구성하느냐에 따라 달라진다.

일반적으로 OS는 하나의 구성 방식으로 만들어지지는 않으며 여러 구성 방식을 조합해서 만들어진다.

### 모놀리식

일부 시스템 프로그램과 커널로 구성되는 방식으로 핵심인 커널은 서비스에 대한 인터페이스와 장치 드라이버로 구성된다. 가장 단순한 구조이다.

시스템 콜 그리고 하드웨어 인터페이스 사이의 모든것이 커널에 결합되어 있으며 커널 내에 모든 기능이 있으므로 시스템 콜에 대한 오버헤드가 없고 커널 내의 통신 속도가 빠르다. 대신 확장이 어렵다는 단점이 있다.

### 레이어드

OS를 사용자부터 가장 밑의 하드웨어까지 여러 층으로 나누어 구성하는 방식이다.

각 층은 자신의 하위층의 서비스와 기능만 사용하도록 구성한다. 특정 계층의 기능이 올바르게 구현되었다면 해당 계층을 사용하는 계층에서는 자신의 계층만 생각하면 되므로 오류 발견이 쉽다. 대신 사용자 계층에서 하드웨어 계층까지 여러 계층을 통과해야 하므로 오버헤드가 발생한다.

### 마이크로 커널

일부 중요한 구성요소를 제외하고 모두 커널에서 분리하는 방식이다. 커널에서 분리할 대상이 명확한 것은 아니다.

커널에서 모든것을 다 처리하는 모놀리식과는 다르게 대부분의 서비스는 사용자 프로세스로 수행되며 서비스를 제공받고자 하는 클라이언트 프로그램은 커널을 통해서 간접적으로 서비스와 상호작용한다.

각 서비스가 별도의 프로세스로 돌아가기 때문에 하나의 서비스에 문제가 생겨도 다른 서비스에는 영향이 가지 않는 장점이 있다. 대신 서비스를 제공받기 위해 커널을 통해 간접적으로 메시지를 주고 받아야 하므로 오버헤드가 발생한다.

### 모듈

커널이 핵심 구성요소를 가지며 부팅 또는 실행 중 서비스를 모듈을 통해 사용하는 방식이다.

커널 자체에 기능이 추가된 것이 아니라 동적으로 커널과 서비스를 연결하기 때문에 새로운 기능의 추가가 쉽고 통신을 위한 메시지 전달이 필요없다는 장점이 있다.

잘 변하지 않는 핵심 기능은 커널이 제공하며 추가적인 서비스는 모듈로 구현할 수 있다.

### 하이브리드

다양한 구조를 결합하여 사용하는 형태로 현대 OS는 이 방식을 채택한다.
