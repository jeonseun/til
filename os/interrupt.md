# 인터럽트

인터럽트는 CPU가 특정 기능을 처리하는 도중에 기능을 처리해야할 때 사용하는 기능이다.

예를 들어 장치 컨트롤러가 장치 드라이버에 작업 완료등을 알리는 방식이 인터럽트다.

> 장치 드라이버는 컴퓨터 시스템을 구성하는 장치 컨트롤러에 대한 인터페이스를 제공하는 OS의 구성요소중 하나다.

## 인터럽트 구분

인터럽트는 크게 하드웨어 인터럽트, 소프트웨어 인터럽트로 구분할 수 있다.

- 하드웨어 인터럽트
  - I/O 완료, 타이머 경과 등에 의해 발생한다.
  - 하드웨어적 변화에 의해 나타나므로 비동기적이다.
- 소프트웨어 인터럽트
  - 프로그램 내부에서 발생하는 인터럽트로 트랩이라고도 한다.
  - 잘못된 명령어에 의한 예외
  - 시스템 콜
  - 프로그램의 일정한 시점에 발생하므로 동기적이다.

## 인터럽트 처리

인터럽트의 처리과정은 다음과 같다.

1. 특정 조건(I/O 완료, 예외 등)에 의해 인터럽트가 발생한다.
2. CPU는 작업을 멈추고 특정 위치로 실행을 옮긴다.
3. 인터럽트 서비스 루틴을 실행한다.
4. 서비스 루틴 종료 후 CPU는 이전 연산은 재개한다.

인터럽트 서비스 루틴(ISR)은 인터럽트 발생 시 실행되는 특정 코드를 말한다. 인터럽트 발생 시 실행이 이동하는 특정 위치에는 ISR에 대한 시작 주소가 있다.

ISR을 처리하고 나면 이전의 연산으로 돌아가야 하므로 ISR에는 CPU의 현재 연산에 대한 상태를 저장하는 작업이 포함된다.

## 인터럽트 구현 방식

### CPU 하드웨어 

CPU에는 인터럽트 요청 라인이라는 선이 있으며 CPU가 하나의 명령어를 완료할 때마다 이 선을 확인하여 인터럽트 발생여부를 확인한다.

인터럽트 요청 라인은 보통 두개가 있으며 하나는 마스크 불가능 인터럽트 라인, 다른 하나는 마스크 가능 인터럽트 라인이다. 마스크라 함은 인터럽트의 처리를 중단하거나 지연시키는 것을 말한다.

메모리 오류 등과 같은 일부 이벤트가 마스크 불가능 인터럽트에 포함된다.

### 인터럽트 벡터, 인터럽트 체인

빈번하게 발생하는 인터럽트의 빠른 처리를 위해 인터럽트 벡터가 사용된다. 인터럽트 벡터는 ISR의 시작 주소를 모아둔 배열로 해당 인터럽트의 번호를 인덱스로 사용한다.

단 인터럽트 벡터가 표현할 수 있는 개수보다 많은수의 ISR이 있으므로 일반적으로 인터럽트 체인을 사용해서 구현하게 된다. 인터럽트 벡터에는 각 ISR 리스트의 헤더가 들어가며 인터럽트 발생 시 처리가능한 ISR을 찾을 때 까지 리스트를 순회한다.

### 인터럽트 우선순위

인터럽트마다 중요도가 다를 수 있으므로 우선순위를 부여해 중요한 인터럽트를 먼저 처리할 수 있어야 한다.

인터럽트 기법에는 우선순위 레벨을 구현하여 인터럽트 마스킹을 하지 않고도 우선순위에 따라 인터럽트를 처리할 수 있도록 한다.
