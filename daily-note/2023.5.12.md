1. 자바 중첩 클래스

## 자바 중첩 클래스

- 중첩 클래스(nested class)는 다른 클래스 안에 정의된 클래스를 말함
- 중첩 클래스가 아닌 클래스를 최상위 클래스(top level class)라함
- 중첩 클래스를 포함하는 클래스를 편의상 외부 클래스라함
- 중첩 클래스는 `static` 키워드 사용유무에 따라 두가지로 구분됨
  - 내부 클래스(inner class)는 `static` 키워드가 붙지 않은 중첩 클래스
  - 정적 중첩 클래스(static nested class)는 `static` 키워드가 붙은 중첩 클래스

다음은 중첩 클래스의 일반적인 형식

```java
// 최상위 클래스(top level class)
class Outer {

    // 내부 클래스(inner class)
    class Inner { ... }

    // 정적 중첩 클래스(static nested class)
    static class StaticNested { ... }
}
```

- 중첩 클래스는 해당 클래스를 감싸고 있는 외부 클래스의 멤버임

### 중첩 클래스의 접근 제한

- 내부 클래스는 외부 클래스의 `private` 멤버를 포함한 모든 멤버에 접근할 수 있음
- 정적 중첩 클래스는 `static` 멤버에만 접근할 수 있음
- 중첩 클래스는 `public`, `private`, `protected`, `package-private` 로 접근제한을 설정할 수 있음
- 외부 클래스에서 중첩 클래스에 접근할 때 다음 제약이 걸림
  - 외부 클래스의 인스턴스 멤버는 내부 클래스, 정적 중첩 클래스의 멤버에 접근할 수 있음
  - 외부 클래스의 정적 멤버는 정적 중첩 클래스의 멤버에 접근할 수 있음 (내부 클래스에는 불가능)

### 중첩 클래스 사용의 이점

1. 한 곳에서만 사용되는 클래스를 논리적으로 그룹화 함
    - 특정 클래스가 하나의 클래스에서만 쓰이는 경우 두 클래스를 논리적으로 그룹화 할 수 있음
    - 특정 클래스의 헬퍼 클래스를 중첩 클래스로 묶으면 패키지 구조가 간단해짐
2. 캡슐화를 강화함
    - 예를 들어 최상위 클래스 A, B가 있고 B가 A의 `private` 멤버에 접근해야 하는 경우 B를 A의 중첩 클래스로 만들면 `private` 멤버에 접근할 수 있으면서 외부에는 구현을 숨길 수 있음
3. 가독성과 유지보수성을 높임
    - 중첩 클래스를 사용하면 코드가 사용되는 위치에 더 가깝게 클래스를 배치할 수 있음
