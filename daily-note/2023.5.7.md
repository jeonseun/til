## 자바 기술 개요

자바 기술은 자바 플랫폼과 자바 프로그램이 언어 둘다를 뜻함

- 자바 소스코드는 `.java` 확장자로 끝나는 일반 텍스트 파일에 작성되며 자바 컴파일러에 의해 `.class` 파일로 컴파일됨
- `.class` 파일에는 프로세서가 실행가능한 네이티브 코드는 포함되지 않으며 대신 바이트코드가 포함됨
  - 바이트코드는 JVM이 실행할 수 있는 기계 언어
- 자바 런처 툴에 의해서 JVM이 인스턴스화되며 자바 프로그램이 실행됨
- JVM 자체는 여러 플랫폼용으로 제공되므로 동일한 `.class` 파일을 여러 플랫폼에서 실행할 수 있음
- 일부 JVM 구현체는 성능향상을 위한 추가 단계를 수행함
  - 성능 병목 지점을 찾거나 여러번 사용되는 코드 섹션을 찾는 작업등이 포함됨
- 일반적인 플랫폼과 달리 자바 플랫폼은 다른 플랫폼 위에서 돌아가는 소프트웨어 기반 플랫폼
- 자바 플랫폼은 크게 JVM과 Java API로 구성됨
  - JVM은 바이트코드를 실행하는 주체이며 다양한 하드웨어 기반 플랫폼용으로 제공
  - Java API는 자바 프로그램 개발에 사용되는 유용한 기능을 제공하는 일종의 소프트웨어 기성품
- 플랫폼 독립적인 특징으로 인해 자바는 타 언어에 비해 느릴 수 있으나 컴파일러와 JVM의 발전으로 성능은 많이 개선된 상태

## 자바 Hello World

```java
class HelloWorldApp {
    public static void main(String[] args) {
        System.out.println("Hello World!"); // 문자열을 출력한다.
    }
}
```

위 예제는 코드 주석, 클래스, main 메소드로 구성됨

### 주석

주석은 컴파일러에는 무시되지만 다른 개발자에게 유용한 정보를 남기는데 사용될 수 있음

- `// ~` - 한줄 주석으로 해당 줄의 모든 내용을 주석처리
- `/* ~ */` - /*와 */ 사이의 모든 내용을 주석처리
- `/** ~ */` - javadoc 툴에 의해서 문서를 자동으로 생성할 때 사용하는 주석

### 클래스

클래스 정의의 기본 형식은 다음과 같음

```java
class Name {
    ...
}
```

- 클래스 정의는 `class` 키워드로 시작함
- 클래스의 내용은 `{}` 안에 위치함

### main 메소드

모든 자바 프로그램이 가져야 하는 메소드로 프로그램의 진입점이 됨

```java
public static void main(String[] args) {
    ...
}
```

- `public`, `static` 순서는 바뀌어도 되지만 관례상 `public static`으로 사용함
- 메소드 인수 이름은 자유롭게 선택할 수 있지만 보통 args, argv를 사용함
- 메소드 인수 배열에 담기는 내용은 자바 프로그램 실행시 넘겨주는 인수 값에 해당함
  - `java MyApp arg1 arg2 arg3` 처럼 자바 프로그램을 실행하면 배열에 arg1, arg2, arg3가 담겨짐
  - 이때 각 문자열 인수를 command-line argument라 함

### 일반적인 오류

1. `Exception in thread "main" java.lang.NoSuchMethodError: main`

    main 메소드가 없는 경우 발생

2. `Exception in thread "main" java.lang.NoClassDefFoundError: HelloWorldApp/class`

    자바 런처 툴로 실행할 때 `.class` 확장자를 붙이는 경우 발생
    
    자바 런처 툴은 클래스 이름으로 실행할 프로그램을 결정하므로 파일명인 `xxx.class` 를 입력하면 해당 클래스를 찾지 못한다는 오류가 발생함

3. `Exception in thread "main" java.lang.NoClassDefFoundError: HelloWorldApp`

    자바 런처 툴로 실행할 때 `.class` 파일의 위치를 찾지 못해서 발생하며 클래스패스를 추가하거나 `.class` 파일이 있는 곳에 가서 자바 프로그램을 실행해야 함

## 자바 OOP 개요

### 객체

- 현실 세계의 물체는 **상태**와 **행동**이라는 두 가지 특성으로 설명될 수 있음
- 현실 세계의 물체는 서로 다른 물체를 포함하기도 하며 복잡하게 구성됨
- 소프트웨어의 객체는 현실 세계의 물체와 개념적으로 연결되며 마찬가지로 **상태**와 **행동**을 가짐
  - 객체는 **필드**에 상태를 저장하며 **메소드**를 통해 행동을 표현함
  - 메소드는 객체 내부 상태를 기반으로 동작하며 객체간 통신의 기본 매커니즘
  - 객체 내부의 상태를 숨기고 메소드를 통해서 상호작용 하는것을 데이터 캡슐화라 하며 이는 객체지향의 기본 원칙중 하나
- 소프트웨어 구성요소를 객체라는 단위로 묶는 것은 다음의 이점을 지님
  - 모듈성: 각 객체의 코드를 개별적으로 관리 가능
  - 정보 은닉: 메소드를 통해 상호작용 함으로 객체 내부 구현을 외부에 숨길 수 있음
  - 코드 재사용: 전문 개발자에 의해 만들어진 객체를 자신의 프로그램에 사용할 수 있음
  - 플러거블 및 디버깅 용이: 특정 객체에 문제가 생길 경우 해당 객체만 변경하면 됨

### 클래스

현실 세계에서도 여러 물체가 같은 종류인 경우가 있으며 이를 OOP 관점에서 말하면 동일한 클래스의 인스턴스라 함 여기서 클래스는 개별 객체를 만들기 위한 일종의 설계도가 됨

```java
// 자전거라는 객체에 대한 클래스 예시
class Bicycle {
    int cadence = 0;
    int speed = 0;
    int gear = 1;

    void changeCadence(int newValue) { ... }
    void changeGear(int newValue) { ... }
    void speedUp(int increment) { ... }
    void applyBrakes(int decrement) { ... }
    void printStates() { ... }
}
```

위 클래스는 cadence, speed, gear라는 상태를 필드의 형태로 가지며 changeCadence, changeGear등의 행동을 메소드의 형태로 정의하고 해당 메소드를 통해 객체가 외부와 상호작용 할 수 있게 함

### 상속

- 서로 다른 객체가 일정한 공통부분을 가지는 경우 공통된 특성을 물려받을 수 있도록 지원함 이를 상속이라 함
- 공통된 특성을 물려주는 클래스를 슈퍼 클래스, 물려받는 클래스를 서브 클래스라 함
- 일반적으로 슈퍼 클래스에 대한 서브 클래스의 수는 무제한임

만약 자전거에 이를 적용해 일반적인 자전거가 가지는 특성을 `Bicycle` 이라는 클래스로 정의하고 산악용 자전거를 `MountainBike` 라는 클래스로 정의할 때 자바 문법으로 보면 다음과 같음

```java
class MountainBike extends Bicycle {
    ...
}
```

- `MountainBike` 는 `Bicycle` 의 특성을 그대로 이용하면서 자신의 고유한 특성을 정의할 수 있음
- `MountainBike` 에는 자신의 고유한 특성만 소스코드에 나타내면 되는 장점이 생김
- 단 슈퍼 클래스의 소스코드는 서브 클래스에 표시되지 않으므로 적절히 문서화할 필요가 있음

### 인터페이스

객체는 메소드를 통해 외부와 상호작용 함 달리 말하면 메소드는 객체와 외부와의 인터페이스를 형성한다고 할 수 있음

예를 들어 tv 리모컨의 버튼은 리모컨의 전기적 장치와 사람을 상호작용 할 수 있게 해주는 인터페이스에 해당함

OOP에서도 인터페이스를 지원하며 형태는 다음과 같음

```java
interface Bicycle {
    void changeCadence(int newValue);
    void changeGear(int newValue);
    void speedUp(int increment);
    void applyBrakes(int decrement);
}
```

일반적인 형태는 구현을 가지지 않는 메소드의 선언으로 위 인터페이스는 자전거가 수행할 수 있는 행동 자체를 나타냄

인터페이스는 특정 클래스가 구현할 수 있으며 그 형태는 다음과 같음

```java
class MyBicycle implements Bicycle {
    // 인터페이스의 메소드를 구현
    void changeCadence(int newValue) { ... }
    void changeGear(int newValue) { ... }
    void speedUp(int increment) { ... }
    void applyBrakes(int decrement) { ... }

    // 개별 클래스의 고유한 메소드도 정의할 수 있음
    void printStates() { ... }
}
```

- 인터페이스 자체는 해당 인터페이스를 구현하는 클래스에 대해서 해당 클래스가 외부와 ~한 방식으로 상호작용 할 수 있다는 것을 나타냄
  - 이는 인터페이스에 정의해둔 행동, 즉 메소드는 구현하는 클래스가 구현하기 때문에 얻을 수 있는 이점
  - 인터페이스를 구현하는 클래스는 컴파일러에 의해 인터페이스의 메소드를 구현하는지 확인됨

### 패키지

패키지는 관련있는 클래스와 인터페이스의 모음이며 개념적으로 컴퓨터의 폴더와 비슷함

자바 코드를 작성할 때 수많은 클래스와 인터페이스를 정의하므로 관련성에 맞게 패키지에 배치하는 것이 좋음

자바는 다양한 기능이 있는 Java API를 패키지의 형태로 제공함 이를 통해 개발자가 기계적인 구현 자체가 아닌 만들고자 하는 애플리케이션의 설계에 집중할 수 있게 함

패키지 별로 나뉘어서 제공되는 Java API는 [Java API Specification](https://docs.oracle.com/en/java/javase/19/docs/api/index.html)으로 제공됨

## 변수

- 인스턴스 변수(instance variables, non-static field)
  - 객체는 각자의 상태를 비 정적 필드(`static` 키워드 없이 선언된 필드)에 저장함
  - 비 정적 필드는 클래스의 인스턴스마다(각 객체마다) 고유하므로 인스턴스 변수라고 함
- 클래스 변수(class variables, static field)
  - `static` 키워드로 선언된 필드로 여러 객체가 생성되더라도 오직 하나만 생성되도록 컴파일러에게 지시함
  - 해당 클래스의 모든 객체가 같은 값을 공유하므로 클래스 변수라 함
  - `final` 키워드를 사용하는 경우 변하지 않는 값으로 만드는 것도 가능
- 지역 변수(local variables)
  - 메소드의 일시적인 상태를 저장하는 변수
  - 지역 변수임을 나타내는 별도의 키워드가 없으므로 오직 선언된 위치에 의해 결정됨
  - 지역 변수는 메소드의 여는 괄호와 닫는 괄호 사이에 선언된 변수를 말함
  - 메소드 외부에서 접근할 수 없음
- 매개변수(parameter)
  - 메소드를 호출하는 쪽에서 넘겨주는 것을 받는 변수
  - 매개변수는 필드와는 구분되며 항상 변수로 취급함

### 필드 vs 변수

- 지역 변수와 매개변수를 제외하고는 일반적으로 필드라고 함
- 지역 변수와 매개변수를 포함하는 모든 종류의 변수는 단순히 변수하고 할 수 있으며 명확한 구분이 필요한 경우 지역 변수, 매개변수라 함
- 특정 타입의 필드나 메소드 또는 중첩된 타입의 경우 멤버라고 하기도 함

### 명명 규칙

- 변수명은 대소문자를 구분함
- 변수명에는 $, _, 문자, 숫자가 들어갈 수 있으며 공백은 허용되지 않음
- 변수명은 $, _, 문자로 시작할 수 있음
  - 관례상 문자로 시작하는 것이 일반적
  - $ 기호로 시작하지 않는 것이 권장됨, 자동 생성된 이름등에서 볼 수 있음
  - _ 기호 또한 사용하는 것은 가능하지만 변수명의 시작으로 쓰지 않는 것이 나음
- 변수명에는 약어 대신 풀네임을 쓰는 것이 권장됨
  - 풀네임을 쓰는 것은 코드를 읽고 이해하기 쉽게 만듬
  - 이 경우 코드 자체가 문서의 역할을 할 수 있게 도움
- 변수명에는 키워드나 예약된 단어는 사용되지 않아야 함
- 변수명에 들어가는 단어가 하나일 경우 모두 소문자로 함
- 변수명에 하나 이상의 단어가 포함되는 경우 두번째 단어부터 첫글자를 대문자로 함
- 상수를 저장하는 경우 모든 단어를 대문자로 하며 여러 단어일 경우 _ 로 연결함
  - 이 경우를 제외하면 변수명에 _ 는 쓰지 않음

### 기본 데이터 타입

자바는 정적 타입 언어로 사용되기 전에 타입이 지정되어야 함 이 과정에는 변수의 타입을 지정하고 변수명을 결정하는 작업이 포함됨

- 자바는 8가지의 기본 타입을 제공하며 이 타입의 이름은 키워드로 예약되어 있음
- 기본 타입의 변수는 다른 변수와 값을 공유하지 않음

자바의 기본타입

- `byte`
  - 8-bit 부호있는 2의 보수 형식 정수를 나타냄 범위는 -128 ~ 127
  - 메모리 절약이 중요한 경우 `byte` 타입은 유용할 수 있음
  - 타입에 의한 크기 제한이 코드를 더욱 명확하게 만드는 경우 `int` 대신 사용할 수 있음
- `short`
  - 16-bit 부호있는 2의 보수 형식 정수를 나타냄 범위는 -32,768 ~ 32,767
  - `byte` 와 동일한 지침 적용
- `int`
  - 32-bit 부호있는 2의 보수 형식 정수를 나타냄 범위는 $-2^{31} \sim 2^{31} - 1$
  - Java 8 이상 부터는 $0 \sim 2^{32} - 1$의 범위를 가지는 unsigned 형을 사용가능
- `long`
  - 64-bit 부호있는 2의 보수 형식 정수를 나타냄 범위는 $-2^{63} \sim 2^{63} - 1$
  - `int` 보다 넓은 범위를 써야하는 경우 사용할 것
  - Java 8 이상 부터는 $0 \sim 2^{64} - 1$의 범위를 가지는 unsigned 형을 사용가능
- `float`
  - IEEE 754 표준의 단정밀도 32-bit 부동소수점
  - `byte` 와 같은 맥락으로 `double` 에 비해 더 적은 메모리 공간을 사용함
  - 부동소수점 방식은 반올림을 통해 근사치를 사용하므로 매우 정확한 계산이 필요한 경우 사용해서는 안됨
- `duoble`
  - IEEE 754 표준의 배정밀도 64-bit 부동소수점
  - 실수값에서의 기본값
  - `float` 와 마찬가지로 정확한 계산이 필요한 경우 사용해서는 안됨
- `boolean`
  - `true`, `false` 의 두 값만을 가짐
  - 참/거짓의 논리값을 나타내기 위해서 사용됨
  - 1bit로 나타낼 수 있는 정보이나 정책에 따라 크기가 달라질 수 있음
- `char`
  - 16-bit 유니코드 문자를 나타냄
  - 범위는 0 ~ 65,535 (\u0000 ~ \uffff)

문자열 데이터 타입

- 문자열을 나타내기 위해 자바는 `String` 타입을 지원함
- 문자열을 큰따옴표로 묶으면 `String` 객체를 생성함
- `String` 타입은 불변으로 한번 생성되면 변경할 수 없음
- `String` 은 기술적으로는 기본 데이터 타입은 아니지만 자바에서 제공하는 여러 특징에 의해 기본 데이터 타입처럼 취급함

기본값 할당

변수의 선언이후 별도로 기본값을 할당하지 않아도 컴파일러에 의해 기본값이 설정됨 (물론 컴파일러에 의한 기본값에 의존하는 것은 좋은 방식이 아님)

- `byte`, `short`, `int`, `long` - 기본값 0
- `float` - 기본값 0.0f
- `double` - 기본값 0.0d
- `char` - 기본값 '\u0000'
- `String`(+모든 참조 타입) - 기본값 null
- `boolean` - 기본값 false

지역 변수의 경우 초기값을 할당하지 않으면 컴파일 오류임

리터럴

리터럴은 소스코드에 표현되는 고정된 값을 말함

정수 리터럴

- 정수 리터럴은 L 또는 l로 끝나지 않은 경우 모두 `int` 로 간주함
- 정수 리터럴은 10진수, 16진수, 2진수로 표현가능
  - 정수 리터럴의 앞에 아무런 표시가 없는 경우 10진수
  - 앞에 0x가 붙는 경우 16진수
  - 앞에 0b가 붙는 경우 2진수

부동소수점 리터럴

- 부동소수점 리터럴은 F 또는 f로 끝나는 경우 `float` 타입이며 D 또는 d로 끝나거나 아무것도 붙지 않는 경우 `double` 타입임
- 부동소수점 리터럴은 E 또는 e를 사용하는 과학적 표기법으로 나타낼 수 있음
  ```java
  // d1과 d2는 같은 값의 다른 표현
  double d1 = 123.4;
  double d2 = 1.234e2;
  ```

문자 및 문자열 리터럴

- `char` 와 `String` 리터럴은 유니코드 문자(UTF-16 으로 인코딩)를 포함할 수 있음
- 에디터 등에서 유니코드 문자 사용이 불가능한 경우 '\uAC00'(한글 가) 처럼 유니코드 이스케이프를 통해 나타낼 수도 있음
- 작은따옴표로 감싸진 문자는 `char` 타입의 문자 리터럴
- 큰따옴표로 감싸진 문자는 `String` 타입의 문자열 리터럴
- 일부 특수한 이스케이프 시퀀스도 있음(`\\` - `\`, `\t` - 탭, ...)

`null` 리터럴

- `null` 리터럴은 참조 타입에 사용할 수 있음
- `null` 은 기본 데이터 타입을 제외한 모든 타입에 할당될 수 있음
- `null` 은 존재를 확인하는 것 이외에는 할 수 있는 것이 거의 없음
- `null` 은 객체가 사용불가능하다는 것을 나타내는 마커로 종종쓰임

클래스 리터럴

클래스의 이름뒤에 `.class` 를 붙여서 클래스 리터럴에 접근할 수 있으며 타입 자체를 나타냄

숫자 리터럴에 _ 사용

- Java 7 이상부터는 숫자 리터럴에 _ 를 사용할 수 있음
- 숫자 리터럴 단위별로 구분해서 가독성을 높일 수 있음
- 예시
  ```java
  long creditCardNumber = 1234_5678_1234_1512L;
  long securityNumber = 999_99_9999L;
  float pi = 3.14_15F;
  long hexBytes = 0xFF_EC_DE_5E;
  long hexWords = 0xCAFE_BABE;
  long maxLong = 0x7fff_ffff_ffff_ffffL;
  byte nybbles = 0b0010_0101;
  int bytes = 0b11010010_01101001_10010100_10010010;
  ```
- _ 는 숫자 사이에는 넣을 수 있으며 다음 경우에는 불가능
  - 숫자 리터럴의 시작과 끝
  - 부동소수점 리터럴의 소수점에 인접
  - F 나 L 등의 접미사 앞
  - 숫자 문자열
