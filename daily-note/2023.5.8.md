## 자바 배열

- 배열은 고정된 개수의 단일 타입 객체를 담을 수 있는 컨테이너 객체
- 배열의 크기는 생성 시 결정되며 이후 변경할 수 없음
- 배열의 각 아이템은 요소(element)라고 함
- 각 요소는 숫자 인덱스를 통해 접근할 수 있음
  - 인덱스는 0부터 시작함
  - 배열의 길이가 10인 경우 0 ~ 9가 인덱스가됨

```java
int[] arr;

arr = new int[10];
arr[0] = 100;
arr[1] = 200;
...
arr[9] = 1000;

System.out.println(arr[0]);
System.out.println(arr[1]);
...
System.out.println(arr[9]);
```

일반적인 프로그래밍 상황에서는 배열 요소를 접근할 때 반복문을 사용함

### 배열 참조 변수

배열 객체를 참조하는 변수를 선언하는 문법은 다음과 같음

```java
// 타입[] 변수명;
// 배열의 크기는 배열 참조 변수의 선언부에는 들어가지 않음
int[] arr;
```

배열의 선언은 배열 객체를 생성하지 않으며 해당 변수가 배열에 대한 참조라는 것만 컴파일러에게 알림

### 배열의 생성, 초기화 및 요소 접근

1. `new` 연산자를 이용한 배열 생성
  ```java
  int[] arr = new int[10];
  ```
2. 배열 인덱스를 통한 값 초기화
  ```java
  arr[0] = 100;
  arr[1] = 200;
  ...
  arr[9] = 1000;
  ```
3. 배열 인덱스를 통한 요소 접근
  ```java
  System.out.println(arr[0]);
  ...
  ```

배열 생성과 초기화는 다음과 같은 축약형으로 사용할 수 있음

```java
int[] arr = {100, 200, 300, 400, 500, 600, 700, 800, 900, 1000};
```

이 경우 배열의 길이는 중괄호 사이의 요소의 개수로 결정되며 중괄호 사이의 ,로 구분된 값들이 배열의 요소가 됨

### 다차원 배열

- 배열 선언 시 여러개의 []를 사용할 경우 배열의 배열을 사용할 수 있음
- 일반적으로 다차원 배열이라고 함
- 자바의 다차원 배열은 배열의 각 요소가 배열임
- 다차원 배열은 각 요소에 해당하는 배열의 길이가 서로 다를 수 있음

```java
String[][] names = {
    {"Mr.", "Mrs.", "Ms."},
    {"Smith", "Jones"}
}

System.out.println(names[0][0] + nanes[1][0]);
System.out.println(names[0][2] + nanes[1][1]);
```

### 배열의 길이

배열의 길이는 내장 프로퍼티인 `length` 를 통해서 얻을 수 있음

```java
int[] arr = new int[10];
// 10을 출력함
System.out.println(arr.length);
```

### 배열 복사

- 할당 연산으로는 배열이 복사되는 것이 아닌 같은 참조값만 가지게 되므로 같은 배열에 접근하는 것이 됨
- 배열 복사를 위해서는 새로운 배열을 만들고 복사할 대상에 원본 배열의 요소와 같은 요소를 넣어주는 작업이 필요
- 자바에서는 `System` 클래스에서 배열 복제를 위한 메소드를 제공함
  - `public static void arraycopy(Object src, int srcPos, Object dest, int destPos, int length)`
  - src: 원본 배열
  - srcPos: 복사할 시작 인덱스
  - dest: 내용이 복사될 배열
  - destPos: 복사될 시작 인덱스
  - length: 복사할 개수

```java
String[] copyFrom = {"a", "b", "c", "d", "e"};
String[] copyTo = new String[5];

System.arrcycopy(copyFrom, 0, copyTo, 0, 5);
```

### 배열 조작

- 배열 복사의 `arrarcopy` 처럼 자바는 편의성을 위한 배열 조작 메소드를 제공함
- 배열 조작을 위한 메소드는 `java.util.Arrays` 클래스에서 제공됨
  - 배열 복사를 위한 또 다른 메소드로 `copyOfRange` 메소드를 제공함
  - `System.arraycopy` 와의 차이는 `copyOfRange` 는 복사된 배열 자체를 반환함
  ```java
  import java.util.Arrays;
  ...
  String[] copyFrom = {"a", "b", "c", "d", "e"};
  String[] copied = Arrays.copyOfRange(copyFrom, 0, 3);
  ```
  - 첫번째 매개변수는 복사할 대상
  - 두번째 매개변수는 복사를 시작할 인덱스
  - 세번째 매개변수는 복사를 끝낼 인덱스
  - 따라서 `Arrays.copyOfRange(copyFrom, 0, 3);`는 copyFrom의 인덱스 0번부터 2번까지를 복사한 배열을 반환하게 됨
- `binarySearch` 특정값을 가진 요소를 찾는 메소드
- `equals` 두 배열의 같은지 확인하는 메소드
- `fill` 특정 값으로 배열을 채우는 메소드
- `sort` 배열의 요소를 오름차순으로 정렬하는 메소드
- `parallelSort` 배열의 요소를 오름차순 정렬하되 병렬처리하는 메소드
- `stream` 배열로 스트림 API를 쓸 수 있게 하는 메소드
  ```java
  Arrays.stream(copyTo).map(alpha -> "[" + alpha + "]").forEach(System.out::println);
  ```
- `toString` 배열의 요소를 보기좋은 문자열로 만들어 주는 메소드

## 자바 연산자

- 연산자는 1 ~ 3개의 피연산자에 대해 특정 작업을 수행한 후 결과를 주는 특수한 기호임
- 연산자끼리는 우선순위를 가지며 우선순위가 높은 연산자가 먼저처리됨
- 연산자 우선순위가 같은 경우 먼저 나온 연산자가 먼저처리됨
- 할당 연산자를 제외한 모든 연산자는 왼쪽에서 오른쪽으로 처리되며 할당 연산자는 오른쪽에서 왼쪽으로 처리됨

연산자 우선순위

1. `expr++`, `expr--` 후위 연산자 
2. `++expr`, `--expr`, `+expr`, `-expr`, `~`, `!` 단항 연산자
3. `*`, `/`, `%` 곱셈 연산자
4. `+`, `-` 덧셈 연산자
5. `<<`, `>>`, `>>>` 시프트 연산자
6. `<`, `>`, `<=`, `>=`, `instanceof` 관계 연산자
7. `==`, `!=` 동등 연산자
8. `&` 비트 연산자 AND
9. `^` 비트 연산자 XOR
10. `|` 비트 연산자 OR
11. `&&` 논리 연산자 AND
12. `||` 논리 연산자 OR
13. `? :` 삼항 연산자
14. `=`, `+=`, `-=`, `*=`, `/=`, `%=` ,`&=`, `^=`, `|=`, `<<=`, `>>=`, `>>>=` 할당 연산자

### 할당 연산자

변수에 값을 할당하기 위해 사용되는 연산자로 우측의 값을 좌측에 변수에 할당함

```java
// 변수에 할당
int speed = 1;

// 참조 변수에 객체 참조를 할당하는 것도 가능
Object obj = someObj;
```

### 산술 연산자

덧셈, 뺄셈, 곱셈, 나눗셈, 나머지 연산

- `+` 덧셈 연산자 (문자열 접합에도 사용 가능)
- `-` 뺄셈 연산자
- `*` 곱셈 연산자
- `/` 나눗셈 연산자
- `%` 나머지 연산자

```java
System.out.println(1 + 2);
System.out.println(10 - 2);
System.out.println(5 * 2);
System.out.println(11 / 2);
System.out.println(11 % 2);
```

산술 연산자와 할당 연산자를 축약해서 사용하는 것도 가능함

```java
int num = 10;

// num에 5를 더함
num += 5; // num = num + 5;와 동일
```

`+` 연산자는 문자열을 이어붙이는데도 쓸 수 있음

```java
String first = "This is";
String second = " a concatenated string.";
String concatenated = first + second;
System.out.println(concatenated);
```

### 단항 연산자

단항 연산자는 하나의 피연산자에 대해서만 쓸 수 있음

- `+` 양수 임을 타나내는 연산자, 없는 경우 숫자는 양수 취급
- `-` 음수 임을 나타내는 연산자
- `++` 증가 연산자, 숫자 1을 증가 시킴
- `--` 감소 연산자, 숫자 1을 감소 시킴
- `!` 논리 반전 연산자, `boolean` 값을 반전 시킴

증가 연산자와 감소 연산자는 전위, 후위로 사용할 수 있음

```java
int num = 0;
num++;
++num;
num--;
--num;
```

- 전위 연산과 후위 연산의 결과는 똑같이 1을 증가 시키거나 1을 감소시킴
- 차이점은 `++num` 은 num을 1 증가 시킨 값으로 처리하는 하며 `num++` 은 아직 1이 증가되지 않은 값으로 처리함
- 일반적으로 값을 1 증감만 시키는 경우 전위, 후위를 고르는 차이는 없으나 더 복잡한 표현식에서는 차이를 만들 수 있으므로 주의해야 함

### 동등, 관계, 조건 연산자

동등 연산자와 관계 연산자는 두개의 피연산자의 크기 비교 또는 동등 비교를 수행함

- `==` 두 피연산자가 같은지 비교, 할당 연산자 `=` 와 다름 주의
- `!=` 두 피연산자가 다른지 비교
- `>` 더 큰지 비교
- `>=` 크거나 같은지 비교
- `<` 더 작은지 비교
- `<=` 작거나 같은지 비교

조건 연산자 `&&` 와 `||` 는 두 `boolean` 식에 대한 조건부 AND, OR 연산을 수행함

- `&&` 조건부 AND 연산자
- `||` 조건부 OR 연산자
- 두 연산자는 필요한 경우에만 두번째 피연산자를 평가함

삼항 연산자

- 삼항 연산자는 3개의 피연산자를 받음
- 삼항 연산자 `? :` 는 if-then-else 구문의 축약형으로 볼 수 있음
- `exp1 ? exp2 : exp3` 라는 식은 exp1이 참일 경우 exp2로 거짓일 경우 exp3를 실행함
- 별도의 할당 연산자나 복잡한 경우가 아니라면 삼항 연산자를 써서 코드의 가독성을 높일 수 있음

타입 비교 연산자 `instanceof`

- 객체의 타입을 비교하는 연산자
- 어떤 클래스의 인스턴스인지 비교
- 어떤 클래스의 서브 클래스의 인스턴스인지 비교
- 어떤 인터페이스의 구현 클래스의 인스턴스인지 비교
- 주의할 점은 `null` 인 인스턴스가 아니므로 무조건 연산의 결과로 무조건 `false` 가 나옴

### 비트, 비트 시프트 연산자

비트 연산자는 정수 타입에 사용할 수 있는 연산자로 사용하는 빈도는 비교적 낮음

- `~` 단항 비트 반전 연산자, 모든 자리의 비트를 반전 시킴
- `>>`, `<<` 시프트 연산자, 비트 패턴을 통째로 왼쪽 또는 오른쪽으로 옮김
  - 좌측 피연산자는 옮길 비트 패턴
  - 우측 피연산자는 이동할 양
- `>>>` 부호 없는 시프트 연산자, 이동한 뒤 맨앞의 비트를 0으로 채움
- `>>` 는 좌측 피연산자의 부호에 따라서 맨앞의 비트의 값을 채움
- `&` 비트 AND 연산자
- `^` 비트 XOR 연산자
- `|` 비트 OR 연산자
